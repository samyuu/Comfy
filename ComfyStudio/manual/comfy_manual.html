Opening and Saving Charts

Comfy Studio uses the custom .csfm file format for storing chart data. Charts can be opened by clicking the File > Open... menu option or pressing Ctrl+O to open a file dialog. Chart files can also be opened by being dropped onto the Comfy Studio window or launched via windows file association. The File > Open Recent menu allows quick access to recently opened files and can be cleared using the File > Open Recent > Clear Items menu option. This menu is disable if no previous file has been opened or the list has been recently cleared by the user.

Charts can be saved by clicking the File > Save menu option or pressing Ctrl+S. To save a chart to a new location click the File > Save As... menu option or press Ctrl+Shift+S to open a file dialog. If the Song File Name is specified as an absolute path and the Copy Song to Directory checkbox is checked then the associated audio file will be copied to the directory of the new .csfm chart file and the Song File Name path will be updated to be relative.

The File > New Chart menu option can be used to close the current chart being edited and create an empty new one.
File > Open Chart Directory... will open the directory of the current chart in the Windows File Explorer. This menu option is disable if the chart has not yet been saved to disk.

The current chart file name is displayed in the Comfy Studio window title bar. Attempting to close or otherwise discard a chart while having unsaved changes (indicated by an asterisk next to the chart name in the window title) will prompt the user with a warning dialog.
Clicking the Save Changes button or pressing Enter will save the chart and continue. Clicking the Discard Changes button will discard all pending changes. Clicking the Cancel button or pressing Esc will retain all changes and abort / cancel the previously user made action to have prompted the dialog.

In addition to the .csfm chart format, Comfy Studio supports importing and exporting charts in the .pje format used by UPDC through the File > Import... and File > Export As... menu options respectively. Doing so comes at a loss of precision and perfect compatibility is not guaranteed. This feature is more so intended as a crutch while future import and export options are still being developed.

Loading a Song

The first step in creating a chart is choosing and loading an audio file. This can be done through the Chart Properties window. Either manually enter a file path into the Song File Name input text field, click the ... button to open a file dialog or drop an audio file on top of the Comfy Studio window. Upon initially loading the audio file the chart Song Title and Duration will be set automatically to that of the audio file.

The currently supported audio file formats are:

  • Free Lossless Audio Codec (*.flac)
  • Ogg Vorbis (*.ogg)
  • MPEG Audio Layer III (*.mp3)
  • Waveform Audio File Format (*.wav)

Once the audio file has been successfully loaded an audio waveform will appear in the Target Timeline window.

Setting Metadata

In addition to selecting an audio file, the Chart Properties window can also used for setting additional chart related properties:

  • Song Title

    • The text displayed inside the top part of the gameplay HUD. Defaults to "ダミー" if unspecified.

  • Song Artist
  • Song Album
  • Song Lyricist
  • Song Arranger
  • Extra Info 1 - 4

    • General info about the chosen song and are usually displayed during or prior to the loading screen.
    Extra Info are user-defined custom properties with identical functionality.

  • Chart Creator Name
  • Chart Creator Comment

    • Used for identifying yourself or commenting on something for anyone with access to the chart file.

  • Cover Image
  • Logo Image
  • Background Image

    • Paths to the images displayed in the practice mode background and during the song select menu. These widgets function identically to the Song File Name path widget. Conventionally the image dimensions are 500x500 (1:1) for covers, 860x300 (43:15) for logos and 1280x720 (16:9) for backgrounds. At the moment, images of different sizes will be stretched to fit.

    Image files dropped onto the Comfy Studio window will automatically be assigned if their file name case-insensitively contains at least one of the following keywords:
    Cover: cover, jacket, song_jk, folder, album
    Logo: logo, song_logo
    Background: background, song_bg, haikei

  • Difficulty Type
  • Difficulty Level

    • The difficulty rating displayed in the gameplay HUD and used for sorting during the song select menu.

  • Button Sound
  • Slide Sound
  • Chain Slide Sound
  • Slider Touch

    • The default button sounds associated with the chart and the sounds used inside the editor as well as during playtesting.

Sync and Tempo

Choosing the right tempo and song offset is essential for any chart. The Sync Window is the main method of setting a song's tempo, start offset and duration.

The Start Offset widget is used for setting the offset of the song audio file relative to the start of the chart. A negative offset results in the song starting later, a positive offset earlier. The Target Timeline waveform can be a very useful tool in finding a suitable offset.
The Move by Beat buttons add or subtract a fraction of the duration of a beat at the initial tempo to the Start Offset. This is especially useful when for example delaying the start offset of a song up to the second 4
4 bar as is recommended.
The Duration widget is used for setting the duration of the chart. Upon initially loading an audio file the duration is automatically set to its duration and can also be changed through the Set Song End Target Timeline context menu option.

The Tempo and Time Signature widgets operate on the current Target Timeline cursor position. If the cursor does not sit on an existing Tempo Change, a new one is inserted automatically.
Pressing the Remove button removes the tempo change at the cursor position, the very first tempo change however can not be removed.
Allowed tempos range from 30.00 BPM to 960.00 BPM and both the numerator and denominator of the time signature may range from 1 to 64.

Alternatively Right Mouse click on an existing Tempo Change in the Target Timeline to open an edit popup with similar functionality to the Sync Window Chart Rhythm widgets.

One way of determining a song's tempo is through the BPM Calculator window. Tap along with every perceived beat as you listen to your song to calculate its tempo.
To enter a tap either click the Tap button or press Space while the window is focused. To reset the measurement click the Reset button or press Esc.

By Right Mouse clicking a context menu can be opened with options for:

  • Auto Reset Enabled to automatically reset after 2 seconds of inactivity.
  • Play Tap Sound to enable a metronome tick sound when inputting a tap.
  • Use Bar Tap Sound use the metronome bar instead of the beat sound.
  • Apply to Tempo Map to automatically apply the calculated Running BPM to last tempo change at the cursor position.

Placing Targets

Placing targets in-sync to the music is the very essence of a chart and is done through the Target Timeline window.

Left Mouse click while hovering the timeline area to position your timeline cursor underneath your mouse cursor.
Hold down Left Mouse inside the upper part of the timeline to begin cursor scrubbing. The current cursor time is displayed in the bottom left of the timeline. Pressing Esc will center the cursor. Pressing the Left and Right keys will move the cursor by a single grid step in the corresponding direction and holding down Shift while doing so increases the cursor step distance.

The timeline can be scrolled left and right by using the Mouse Wheel. Holding down Shift increases the scroll speed. Holding down Middle Mouse allows scroll gripping of the timeline. The timeline scrollbar can also be used to quickly navigate the entire timeline.

The timeline can be zoomed in and out by holding down Alt while Mouse Wheel scrolling up and down. The timeline zoom slider can also be used for controlling the zoom level. Holding down Ctrl before clicking on a slider widget turns it into an input text field.

Click the Toggle Playback button or press Space to toggle playback. Click the Stop Playback button or press Esc during playback to not only pause playback but also reset the cursor to the last time point prior to playback (indicated by the white marker line). Mouse Wheel scrolling during playback not only scrolls the timeline, but also seeks through the chart.

The grid division controls the number of sub divisions in a 4
4 bar that the cursor is allowed to move in.
The available division level ranges from 1/4th (1 beat) up to 1/192nd, however divisions more precise than 1/32nd should almost never be used and are often an indication for an inaccurately set tempo or song offset.
The current grid division is displayed in the bottom left of the timeline next to the cursor time display and can be changed by Left Mouse / Right Mouse clicking the division widget, pressing Up / Down, clicking Mouse X1 / Mouse X2 or by clicking one of the Grid Division context menu options.

Targets can be inserted at the current cursor position by pressing:

  • W or I for Triangle
  • A or J for Square
  • S or K for Cross
  • D or L for Circle
  • Q or U for Slide Left
  • E or O for Slide Right
Pressing the corresponding target button a second time will remove the existing target. In case of slides, pressing a second time will insert an additional same directional slide instead which can then be removed by pressing a third time.

Range selections can be made by pressing Tab to mark a start point, moving the cursor and pressing Tab a second time to mark an end point. Range selections are displayed as a white box with the same height as your cursor.
To insert a Chain Slide, first range-select a region, hold down Shift and press the corresponding target slide key: Q / U for left chain slides and E / O for right chain slides.
The default chain slide fragment interval is set to 1/32nd and can be changed through the Chain Slide Division context menu options.
Holding down Shift and pressing a non-slide button key will fill in the range-selection at the current grid division level with normal targets instead.

Targets can be selected by holding down Right Mouse and enclosing them with a selection box. Holding down Shift allows selecting new targets and holding down Alt allows deselecting old targets without clearing the previous selection.

Selected targets can be moved left and right by mouse hovering over them, clicking Left Mouse and dragging them left and right. In the same way, holding down Shift before clicking Left Mouse allows changing the type of selected targets by dragging them up and down instead.
Selected targets can not be moved past, nor be overlapped with each other this way. Sync target pairs also cannot be separated from each other and can only be moved if all targets within a sync pair are selected. To separate sync targets and or move targets past each other the Cut and Paste commands can be used instead.

The upper part of the left side of the timeline contains 7 buttons:

  • Jump to First Beat to position the cursor at the start of the chart.
  • Go to previous Grid Tick to step back the cursor by one grid tick, same as pressing Left.
  • Toggle Playback to toggle playback, same as pressing Space.
  • Stop Playback to stop playback, same as pressing Esc.
  • Go to next Grid Tick to step forward the cursor by one grid tick, same as pressing Right.
  • Jump to Last Beat to position the cursor at the end of the chart.
  • Timeline Settings to open the timeline settings popup.

The timeline settings popup allows individual control of the volume levels for the song and button sounds. The Default Volume buttons reset the audio levels back to the default 100%.
An optional metronome can also be enabled here which will match the chart's time signature and tempo during playback.
These settings only apply within the editor and are not stored or in some way applied to the chart file itself.

Right Mouse clicking while not moving the mouse opens a context menu:

  • Grid Division allows setting the current grid division.
  • Chain Slide Division allows setting the chain slide division used while placing chain slides.
  • Playback Speed allows changing the in-editor song playback speed. Press Comma / Period to quickly speed up or slow down playback.
  • Set Song End sets the Duration of the chart equal to the current cursor position.
  • Cut, Copy and Paste expose cut, copy and paste functionality and can also be accessed by pressing Ctrl+X, Ctrl+C and Ctrl+V.
  • Select All and Deselect will either select or deselect all targets in the entire chart.
  • Refine Selection allows performing selection refinements to the current target selection.
    Refine Selection > Select every Nth Target refines the selection based on an even interval, especially useful for quickly creating multi type target patterns.
    Refine Selection > Shift Selection Left / Right shifts the selection into the specified direction.
    Refine Selection > Select all Single Targets refines the selection to only include targets that don't share the same time point with any other target.
    Refine Selection > Select all Sync Targets refines the selection to only include targets that share the same time point with at least one other sync target.
    Refine Selection > Select all partially selected Sync Pairs refines the selection to also include every target in a sync pair if at least one target within is already selected.
  • Remove Targets removes all selected targets and can also be accessed by pressing Del.

Positioning Targets

The Target Preview window is the main method for editing and previewing target positions, angles and other button path properties.
It has two modes of operation: the Position Tool and the Path Tool which can be switched between by either pressing 1, 2 or using their respective context menu options.

The Target Preview window displays all targets at the current timeline cursor position for as long as they have been selected, not yet hit or have been hit more recently than the Post Hit Linger duration. Like on the Target Timeline, targets can be box-selected by holding down Right Mouse and enclosing them with a selection box. Shift and Alt likewise allow addition and exclusion from the existing selection.

The Position Tool, indicated by distance marker circles rendered around all selected targets, allows manipulation of target positions.
Clicking Left Mouse while hovering over a selected target allows target selection to be moved with the mouse. Holding down Shift while doing so snaps their positions to the grid.
Clicking Left Mouse while not hovering any selected target will open a Row Position Circle instead. Moving the mouse cursor in a direction relative to the mouse-down start position then positions all selected targets in a matching row formation. Holding Shift increases the row steepness when angled and pressing Alt before clicking Left Mouse allows positioning the target row backwards. Backwards row placement is especially useful for fitting targets around existing sync target formations.
Pressing W, A, S, D or Up, Left, Down, Right moves all selected targets by one step in the corresponding direction. Holding Shift while doing so increases the step distance to that of the grid.
Pressing Q / E moves the timeline cursor to the closest previous / next target or sync target pair.

The Path Tool, indicated by button path curves rendered on top of selected targets, allows manipulation of the target button path properties angle and frequency.
Clicking Left Mouse anywhere in the window allows setting the angles of all selected targets equal to the angle between the mouse cursor and the first selected target. Pressing Alt before clicking Left Mouse sets the angle between the mouse cursor and last selected target instead.
Holding down Shift snaps the mouse angle to 15.0° and holding down Alt snaps it to 0.1°.
Mouse Wheel scrolling up and down increments or decrements the angle to all selected targets by a uniform amount. Holding down Shift sets the scroll increment step to 5.0° and holding down Alt sets it to 0.1°.
Clicking Middle Mouse while not setting an angle displays the selected target angles as a numeric value and renders the target button paths as straight lines.

Right Mouse clicking while not moving the mouse opens a context menu with different options depending on the current tool selection:

  • Position Tool select the Position Tool.
  • Path Tool select the Path Tool.

Position Tool specific:

  • Flip Targets Horizontally
  • Flip Targets Vertically flips the position, angle and frequency of all selected targets horizontally / vertically. Can also be accessed by pressing H / J.
  • Position in Row positions all selected targets in a row starting from the first selected target to the direction of the last selected target, spaced correctly based on each target's beat timing. Can also be accessed by pressing U.
  • Interpolate Positions linearly interpolates the position of all selected targets starting at the first selected target up to and including the last selected target, regardless of each target's beat timing. Can also be accessed by pressing I.

Path Tool specific:

  • Invert Target Frequencies sets the frequency of all selected targets to negative itself effectively flipping the path of the button. Can also be accessed by pressing R.
    This option is crucial for quickly matching a target's button path curve to that of its target row direction.
  • Interpolate Angles Clockwise
  • Interpolate Angles Counterclockwise Linearly interpolates the angles of all selected targets starting at the first selected up to and including the last selected target. Can also be accessed by pressing U / I.
  • Angle Increment Settings configures angle increment settings used by the Apply Angle Increment options.
    Angle Increment Settings > Fixed Step Increment causes the beat distance to be ignored from one target to another. This option is generally not recommended.
    Angle Increment Settings > Apply to Chain Slide Fragments increments angles with each chain slide fragment instead of only the starting piece. This option is generally not recommended.
    The Per Beat, Per Beat (Slope) and Per Target slider widgets control the degree angle to be applied from one target to the next. The Slope increment is used if a target is placed at an angle relative to the last.
    The Set Default and Set Default Small button allow quickly switching between the settings for the two most common used configurations. Sticking to these is generally recommended.
  • Apply Angle Increment Positive
  • Apply Angle Increment Negative increments / decrements the angles to all selected targets from one to the next based on the current Angle Increment Settings. Can also be accessed by pressing F / V.

Always available:

  • Settings in-editor render settings. These options are entirely cosmetic and only apply within the editor.
    • Target Area allows configuring the background and target render behavior. Target Area > Post Hit Linger controls the duration after which already hit targets will be rendered in the Target Preview window. These are measured in Beat Tick units with each Tick representing 1/48th of a beat.
    • Practice Background allows configuring the background render behavior for the practice background.
    • Set Default Editor View sets the background render behavior to its editor default: Practice background disabled with the target grid enabled.
    • Set Default Practice View sets the background render behavior to its practice default: Practice background enabled with the target grid disabled.

Another way of setting target properties is the Target Inspector window. Modifications here are always applied to all selected targets simultaneously.

Labels can be Left Mouse click dragged left and right to apply a uniform increment to the property for all selected targets. Holding Shift / Alt allows for more rough / precise drag control.
Entering a numeric value into one of the text input fields applies the same value to the property for all selected targets.

Currently the preset flag, (exact values for) frequency, amplitude and distance can only be edited through the Target Inspector. The Amplitude and Distance property input field widgets also function as combo boxes from where a list of common values can be selected. These combo boxes can either be expanded by clicking on the arrow button on the right or Right Mouse clicking anywhere inside the input field.

Until a property has been edited by the user, targets have the Use Preset flag enabled. Targets with this flag set automatically follow a predefined position path and react dynamically to being moved or edited on the Target Timeline.
If the Use Preset flag is manually re-enabled for a target it will automatically revert back to its predefined path. Resetting the placement for an entire chart for example can be done quickly by select all targets and enabling this property.

Multiple targets placed at the same time point, so called "Sync Targets", conventionally use specific placement and path formation which can be quickly formed through the Sync Target Presets window.
Comfy Studio differentiates between two type of sync presets: Dynamic presets and Static presets.
Dynamic sync presets are represented as the 6 button at the top of the window and dynamically react and adjust with varying behavior according to the target selection.
Static sync presets on the other hand are predefined fixed list of properties that can be applied to a sync target pair without variations.

The list of static sync presets is intended to be user defined, customization however is not yet implemented.

The sync preset button are enabled once a target selection has been made. Mouse hovering a preset button previews its behavior in the Target Preview window. Pressing a preset button then applies its preset to all selected sync target pairs. Sync presets can be applied to multiple different sync target pairs with a single click however only the first valid pair is previewed inside the Target Preview window.

The dynamic sync presets operate on the concept of a single master target that dictates the layout of the other targets within its sync pair. The master target is chosen by it either being the only target selected within its sync pair or in case of multiple pair selected targets, the target with the highest type priority. Targets in the last row of the Target Timeline, are prioritized first.

The supported dynamic sync presets are:

Vertical Left / Right:
Operates on fixed per-type target Y position spacing, stacked vertically on top of each other. Applicable to sync target pairs of any non-slide type and number combination.

Horizontal Up / Down:
Operates on a fixed X position for each target depending on its type. Applicable to sync target pairs of any type and number combination. When used with sync slide target pairs their resulting preset layout various based on which of the 4 vertically split sections of the screen the master slide target is positioned in.

Square:
Operates on a uniform distance for each target to the corresponding screen corner. Applicable to quad non-slide type sync target pairs.

Triangle:
Operates on fixed X position for each target depending on its type and a uniform height distance to the corresponding screen edge. Applicable to Triangle+Square+Cross and Square+Cross+Circle sync target pairs.

The settings button at the bottom right of the window allows further configuration of the dynamic sync preset behavior.

  • Steep Angles use steeper 35.0° instead of 45.0° angles for the vertical and 10.0° instead of 20° angles for horizontal presets.
  • Same Directional Angles set all angles of the vertical or horizontal presets to the same direction, depending on the master target.
  • Inside Out Angles flip the angles of the square preset by 180.0°, turning them inside out.
  • Elevate Bottom Row raise the position of the bottom row targets for the square and triangle presets.

Playtesting

The playtest mode can be entered by clicking the Playtest > Start from Beginning / Playtest > Start from Cursor menu option or pressing F5 / F6. The Playtest > Autoplay Enabled menu option allows enabling or disabling the Autoplay mode before leaving the editor.

At the moment, only Keyboard and DualShock4™ input is supported. Targets can be hit by pressing W, A, S, D, I, J, K, L for normal targets and Q, E or U, O for slide and chain slide targets. 1, 2, 3, 4, 7, 8, 9, 0 are temporarily bound to hit Triangle+Square+Cross+Circle all at once.
DualShock4™ input uses default bindings with the Left Trigger and Right Trigger hitting Triangle+Square+Cross+Circle all at once.

Right Mouse click can be used to open a context menu. Pressing Space, DS4 Options, or clicking the Pause / Resume context menu options toggles playback. While paused, a "PAUSED" label is displayed in the top right corner and the screen is dimmed.
Pressing Enter, DS4 R3 or clicking the Restart from Reset Point context menu option restarts the playback mode from the last set reset point. Pressing Tab / Shift + Tab, DS4 Touch / DS4 L3 or clicking the Move Reset Point Forward / Move Reset Point Backward context menu options moves the restart point back by 10 seconds / moves the restart point forward to the current playback time.
Pressing F1 or clicking the Autoplay Enabled context menu option toggles the autoplay mode. During autoplay manual input is disabled and an "AUTOPLAY" label is displayed in the top right corner of the screen.
Pressing Esc or clicking the Return to Editor context menu option exits out of the playtest mode and returns to the editor, at the time prior to entering playtest mode.

Remaining Interface

Audio Output

Comfy Studio supports audio output through WASAPI in both shared and exclusive mode.
Shared mode streams have the advantage of allowing multiple applications to use the same audio device but come at the cost of higher audio latency.
Exclusive mode streams on the other hand, as is the default in Comfy Studio, provide the lowest possible latency supported by the audio hardware, though be it with the disadvantage of preventing other applications from using the audio device at the same time.

Low audio latency is not only important for improving the feeling of responsiveness but also for ensuring maximum accuracy while charting. Button sounds previewed in the Target Timeline are mixed directly into the audio stream with sample perfect accuracy regardless of the audio hardware used. Having a high audio latency however may incorrectly give the user the impression that targets should be placed earlier than what really matches the song because of the relative delay between the visuals and audio. Because of this, timing should always be done by ear, not by eye. The timeline waveform too is a useful tool for syncing as it is not affected by audio latency.
Furthermore, to ensure perfect synchronization between visuals and audio, the audio render output is used to directly control the timing of the entire application. This essentially means the effective framerate during playback is tied to that of the audio buffer duration, further encouraging the use of low latency methods.

Like is common in many DAWs to lessen the effect of an exclusive stream, the audio device is automatically close and reopened when the Comfy Studio window loses focus while no audio is playing, meaning playback is paused.
Tabbing out to a different window while inactive for example, should therefore not prevent other applications from outputting audio, all while still allowing the lowest possible latency while actively charting.

The menu option at the top right of the menu bar displays info about and allows control over the audio device state.
The Open Audio Device and Close Audio Device menu options allow explicit control over the open state however usually don't need to be set by the user manually.
The Use WASAPI Shared and Use WASAPI Exclusive menu options allow controlling the current stream share mode. Exclusive mode is the default, if access is denied by Windows however then Comfy Studio will always fall back to shared mode automatically.
Exclusive mode permission for an audio device can be configuring inside the Windows Sound > Playback > "Device Name" > Properties > Advanced menu. If undesired by the user or if it is causing issues on specific hardware, exclusive mode access can be systemwide disabled this way.

Edit Menu

The Edit menu inside the menu bar currently contains two options: Edit > Undo also Ctrl+Z and Edit > Redo also Ctrl+Y.
The undo / redo action history can also be viewed and edited inside the Chart Editor History window.
Un-done actions are grayed out and the latest action is highlighted. Clicking on one of the listed actions undoes / redoes the action history up that point.

Window Menu

The Window menu inside the menu bar most importantly contains the options to Window > Toggle Fullscreen also F11 and Window > Swap Interval to enable or disable VSync.
The Window > ImGui Config menu is used for testing and is not intended for general use.

Test Windows Menu

The Test Windows menu inside the menu bar provides access to various test windows. These might come in handy if something breaks but are otherwise not intended for general use.

Help Menu

The Help menu inside the menu bar allows opening of this manual through the Help > Open User Manual menu option. The Help > About Comfy Studio menu option opens a window containing version and license info.

Upcoming Features

The end of the beta won't mark the end of all updates, I have big ambition, however there are certain things I'd like to get done first before I consider Comfy Studio ready for public release.
These are some of the features being considered in the near or distant future:

  • Customizable static sync presets
  • Customizable inspector property combo box values
  • Customizable controls and other editor settings
  • Advanced circle and arbitrary shape / curve placement tools
  • Visual indication for holds directly inside the editor
  • Playtest hold mechanics with correct macro handling for hold transfers
  • DIVA PV script (.dsc) and full MDATA export including automatic handling of multiple chart export and all the things that go along with it like sprites, DBs and audio files
  • Improved chain slide input methods
  • Additional input methods for Holds and the Frequency, Amplitude and Distance properties
  • Additional chart related properties support

And also but with less certainty:

  • (?) Improved sync preset input methods
  • (?) Chance time input method
  • (?) Per-target user defined custom flags to aid in porting to other games. For example having a "W - Arrow" or "Hold End" flag that could then be used by external tools to more easily create F style charts
  • (?) Per target or per tempo change fly time factor
  • (?) DIVA PV script (.dsc) chart import
  • (?) Automatic Chart analysis for detecting common placement mistakes and other errors
  • (?) Different Chart statistics display, target type distribution, placement heat map, etc.
  • (?) Built-in lyrics editor
  • (?) Built-in event editor for other PV events like stage changes, video display, eventually evolving into something more similar to Edit Mode
